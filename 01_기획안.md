# Caliper Works - 프로젝트 기획안

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | Caliper Works |
| 목적 | 브레이크 캘리퍼 생산 현장에서 직원들이 제품 정보를 조회하는 PWA 웹앱 |
| 사용자 | 생산 현장 작업자 (조회 전용) |
| 데이터 | Firebase Firestore (daehansa-workflow 프로젝트) |
| 사진 | Firebase Storage |
| 비용 | Firebase 무료 티어 (사용자 수 제한 없음) |
| 관리 | 데이터 입력/수정은 기존 AppSheet에서 수행 |

---

## 2. 주요 기능

### 2-1. 제품 검색
- 코드, 모델명, 차종, 부품코드로 검색
- 실시간 필터링 (타이핑하면 바로 결과 표시)
- 전체 제품 리스트 스크롤 조회

### 2-2. 제품 상세 정보
- 제품 사진 (Firebase Storage)
- 기본 정보: 코드, 모델명, 제조사, 시스템, 타입, 니플, 규격
- 하우징/캐리어 번호
- 호환차종, 검색참고 정보

### 2-3. BOM (부속품 목록)
- 피스톤 (최대 3개)
- 씰 (최대 3개)
- 부트 (최대 3개)
- 핀 (최대 2개)
- 핀부트 (최대 3개)
- 각 부품 클릭 시 부품 상세 화면으로 이동

### 2-4. 부품 상세
- 부품 코드, 구분 표시
- 해당 부품을 사용하는 다른 제품 목록 (역방향 조회)
- 상세 스펙 (D mm, H mm 등) - 추후 데이터 추가 예정

### 2-5. 위치 정보
- 완제품 위치
- 하우징 위치
- 캐리어 위치
- 창고 위치
- 고품 위치

## 3. 화면 구조

```
[메인 화면]
  ├── 검색바
  ├── 제품 리스트 (썸네일 + 코드 + 모델명 + 태그)
  │
  └── [제품 상세 화면] (슬라이드로 전환)
        ├── 제품 사진
        ├── 제품 정보 그리드
        ├── BOM 리스트
        │     └── [부품 상세 화면] (슬라이드로 전환)
        │           ├── 부품 코드/구분
        │           ├── 상세 스펙 (추후)
        │           └── 사용 제품 목록
        └── 위치 정보 그리드
```

## 4. 데이터 구조

### Firestore 컬렉션: `products`

현재 Firestore의 `daehansa-workflow` 프로젝트 > `products` 컬렉션에 아래 필드가 저장되어 있습니다:

```
문서 ID: 제품코드 (예: "11011", "12222")

필드:
- 코드: "11011"
- 코드모델명: "11011_아토스 LH"
- 제조사: "현대"
- 시스템: "만도"
- 타입: "1P"
- 니플: "L"
- 규격: "344789"
- 검색창내용: "[H]:140184 22 // [C]:412180"
- 호환차종: "비스토"

BOM 필드:
- 피스톤1: "P514401"
- 피스톤2: ""
- 피스톤3: ""
- 씰1: "RS54"
- 씰2: ""
- 씰3: ""
- 부트1: "RB54"
- 부트2: ""
- 부트3: ""
- 핀1: "RG04"
- 핀2: ""
- 핀부트1: "RP01"
- 핀부트2: ""
- 핀부트3: ""

위치 필드:
- 완제품 위치: "A-1-1"
- 하우징: "140033 20S"
- 하우징 위치: "2-1-4"
- 캐리어: "412033"
- 캐리어 위치: "2-1-4"
- 창고 위치: "-"
- 고품 위치: "구색"

기타:
- 이미지: (URL 또는 빈 값)
- 이미지2: ""
- 제품관련 적요: ""
- 최종업데이트: "2026. 1. 15 오후 7:11:52"
- alt=media: (이미지 URL)
```

### Firebase Storage 사진 경로

제품 사진은 아래 경로에서 자동 탐색합니다:
- `products/{코드}.jpg`
- `products/{코드}.png`
- `products/{코드}.JPG`
- `products/{코드}.jpeg`

## 5. 기술 스택

| 항목 | 기술 |
|------|------|
| 프론트엔드 | HTML + CSS + JavaScript (단일 파일) |
| 데이터베이스 | Firebase Firestore |
| 이미지 저장소 | Firebase Storage |
| 배포 | Firebase Hosting |
| 앱 형태 | PWA (Progressive Web App) |
| 인증 | 없음 (읽기 전용 조회 앱) |

---

## 6. 디자인 방향

- 다크 테마 (제조 현장에서 눈의 피로 최소화)
- 모노스페이스 폰트 (JetBrains Mono) + 한글 (Noto Sans KR)
- 오렌지 액센트 컬러 (#FF6B35)
- 위치 정보는 노란색 강조로 빠르게 식별
- 부품 유형별 색상 구분 (피스톤=오렌지, 씰=파랑, 부트=초록, 핀=노랑, 핀부트=보라)
- 모바일 최적화 (600px 이하)

---

## 7. 배포 절차

1. Firebase CLI 설치: `npm install -g firebase-tools`
2. Firebase 로그인: `firebase login`
3. 프로젝트 초기화: `firebase init hosting`
4. public 폴더에 index.html, manifest.json 배치
5. 배포: `firebase deploy --only hosting`
6. 직원들에게 URL 공유 → 홈 화면에 추가
